## Prompt 4 — Reimbursement Workflow

Implement only the reimbursement module.

### Reimbursement Creation

POST /rest/reimbursements

EMP only.

Request:

{
"title": "...",
"description": "...",
"amount": 2500
}

Behavior:

* employeeId from JWT
* amount > 0
* create reimbursement
* default statuses = PENDING

### Reimbursement Approval

PATCH /rest/reimbursements

Roles:

* RM
* APE
* CFO

Request:

{
"reimbursementId": "...",
"status": "APPROVED"
}

### RM Rules

* Only assigned employees
* Reimbursement must exist
* Not finalized

### APE Rules

* RM approval required first

### CFO Rules

* Can approve/reject any reimbursement

### Final Status Logic

* RM rejected → REJECTED
* APE rejected → REJECTED
* RM approved + APE approved → APPROVED
* Otherwise → PENDING

### Visibility Rules

EMP:

* Own reimbursements

RM:

* Pending reimbursements of direct reports

APE:

* RM-approved reimbursements

CFO:

* APE-approved reimbursements

### Additional Endpoint

GET /rest/reimbursements/:userId

Accessible only when:

* Target user is EMP
* Target EMP is subordinate of requesting RM
